🔬 How the ML Model Works
Understanding the AI pipeline behind InsightX forecasting
STEP 01
📥
Data Collection
Daily sales records are stored in SQLite. Each row contains product ID, date, units sold, price, and day-of-week. Minimum 7 days needed to train.
pandas · SQLite
STEP 02
🔧
Feature Engineering
Raw data is transformed into ML features: rolling averages (7d, 14d), day-of-week encoding, lag features (yesterday's sales), month, season flags.
scikit-learn · numpy
STEP 03
🌲
Model Training
A Random Forest Regressor is trained per product. 100 decision trees vote on the predicted demand. Cross-validation ensures no overfitting.
RandomForestRegressor · joblib
STEP 04
📡
Serve via API
Trained model is saved as a .pkl file. Flask loads it and serves predictions through a REST API endpoint that this frontend calls in real-time.
Flask · joblib · JSON API
🔌 Backend API Integration
How this page will connect to your Flask ML backend in Phase 2
GET /api/forecast/<product_id>
Get AI forecast for a specific product
// JavaScript fetch call (Phase 2) const res = await fetch( '/api/forecast/1?days=7' ); const data = await res.json(); // data = { // product_id: 1, // forecast: [48, 55, 51, ...], // confidence: 0.94, // trend: "up" // }
POST /api/model/retrain
Trigger model retraining with latest data
// Retrain API call (Phase 2) const res = await fetch( '/api/model/retrain', { method: 'POST' } ); const data = await res.json(); // data = { // status: "success", // accuracy: 0.942, // trained_at: "2025-03-30" // }
GET /api/forecast/all
Get forecast summary for all products
// Get all forecasts (Phase 2) const res = await fetch( '/api/forecast/all' ); const data = await res.json(); // data = { // products: [...], // stockout_risk: [...], // model_accuracy: 0.942 // }
POST /api/sales/add
Log today's sales to train the model
// Log sales data (Phase 2) const res = await fetch( '/api/sales/add', { method: 'POST', body: JSON.stringify({ product_id: 1, units_sold: 12, date: "2025-03-30" }) } );
